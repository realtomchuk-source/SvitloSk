# SvitloSk: Система моніторингу графіків відключень

Проект для автоматизації збору, аналізу та відображення графіків відключень електроенергії (ГПВ).

## 🚀 Архітектура системи (Roadmap Vision)

Згідно з планом розвитку, система переходить на багатокомпонентну архітектуру:

1. **Frontend Admin (React + Vite)**: **Data Verification Center** — пульт гарантування точності даних.
2. **Analytics Service (FastAPI)**: Мікросервіс на Python для складної статистики та моніторингу навантаження.
3. **Parser (Python)**: Автоматизований збір та розпізнавання (OCR) графіків.
4. **Supabase (DB & Auth)**: Сховище даних, Audit Logs та автентифікація.
5. **Docker**: Контейнеризація всього стеку.

---

## 🛡 Data Verification Center (Admin Mission)

Адмін-панель SvitloSk переходить до моделі **"Центру верифікації"**, де головним пріоритетом є 100% точність даних:
- **Visual Verification**: Звірка напарсених даних з оригінальними джерелами (зображення/PDF) перед публікацією.
- **Audit Logging**: Кожна зміна в графіку або системі фіксується (Хто? Коли? Що змінив?).
- **Notification Load Monitoring**: Моніторинг щільності підписок для стабільної розсилки сповіщень.
- **Human-in-the-loop**: ШІ пропонує — людина підтверджує.
