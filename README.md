# SvitloSk: Система моніторингу графіків відключень

Проект для автоматизації збору, аналізу та відображення графіків відключень електроенергії (ГПВ).

## 🚀 Архітектура системи (Roadmap Vision)

Згідно з планом розвитку, система переходить на багатокомпонентну архітектуру:

1. **Frontend Admin (React + Vite)**: **Data Verification Center** — пульт гарантування точності даних.
2. **Analytics Service (FastAPI)**: Мікросервіс на Python для складної статистики та моніторингу навантаження.
3. **Parser (Python)**: Автоматизований збір та розпізнавання (OCR) графіків.
4. **Supabase (DB & Auth)**: Сховище даних, Audit Logs та автентифікація.
5. **Docker**: Контейнеризація всього стеку.

---

## 🛡 Data Verification Center (Admin Mission)

Адмін-панель SvitloSk переходить до моделі **"Центру верифікації"**, де головним пріоритетом є 100% точність даних:
- **Visual Verification**: Звірка напарсених даних з оригінальними джерелами (зображення/PDF) перед публікацією.
- **Audit Logging**: Кожна зміна в графіку або системі фіксується (Хто? Коли? Що змінив?).
- **Notification Load Monitoring**: Моніторинг щільності підписок для стабільної розсилки сповіщень.
- **Human-in-the-loop**: ШІ пропонує — людина підтверджує.

## 🛠 Повний план реалізації (Roadmap)

### 1. Архітектура та Інфраструктура
- [ ] **FastAPI мікросервіс**: Окремий сервіс для обробки аналітики.
- [ ] **Docker & Docker-compose**: Локальний запуск всього стеку однією командою.
- [ ] **CI/CD**: GitHub Actions для автоматичної збірки Docker-образів.

### 2. Управління користувачами
- [x] Перегляд списку та пошук за email / UUID.
- [x] Блокування та розблокування користувачів в один клік.
- [x] **Редагування ролей**: Зміна прав доступу (admin/moderator/user) безпосередньо з інтерфейсу.
- [x] **Детальна картка**: Перегляд активних пуш-слотів сповіщень та стартової черги користувача.

### 3. Аналітика та Звіти
- [x] Загальна статистика профілів та дій адміністратора за 24 години.
- [x] **Візуалізація підчерг**: Мінімалістична двоколонкова прогрес-сітка капсул для відображення розподілу підписок за 12 підчергами (1.1 - 6.2).
- [x] **Продуктивність та здоров'я**: Інтегровані панелі для моніторингу швидкості серверного парсингу, об'єму SQLite бази даних, uptime API та успішності циклів.
- [x] **Метрики використання додатку**: Моніторинг відвідуваності ключових PWA-екранів (Головна, Пошук адрес, Кабінет, Архів).
- [x] **Статистика Пуш-повідомлень**: Моніторинг активних PWA-токенів, відправлених сповіщень та відсотка успішної доставки.

### 4. Логи Парсера & Запити адрес
- [x] Перегляд, пошук та зручна фільтрація логів за рівнем критичності (INFO, SUCCESS, WARNING, ERROR).
- [x] **Обробка запитів нових адрес**: Повний інтерфейс перегляду та призначення відповідних підчерг для адрес, яких немає в системі.
- [x] **Центр верифікації графіків**: Перегляд та ручне затвердження/редагування нових напарсених розкладів на сітці перед публікацією.

### 5. Налаштування та Інтеграції
- [x] Збереження та управління GitHub PAT токеном у localStorage для виклику GitHub Actions.
- [x] **Trigger Workflow**: Інтерактивний запуск авто-парсингу через GitHub Actions API.

### 6. Управління контентом & Експорт
- [x] **Announcements**: Створення, управління та публікація важливих оголошень для відображення на біжучій стрічці PWA.
- [x] **Grid Editor**: Зручна сітка ручного редагування графіків із функціями заповнення, очищення та інвертування.
- [x] **PNG Telegram Exporter**: Генерація преміальних Telegram-оптимальних зображень (`734x824`px) з горизонтальними суцільними progress-bar лініями, динамічними мітками переходу світла (зміна стану) та брендуванням.

### 7. Автентифікація та Безпека
- [x] **Supabase Auth**: Вхід за email/паролем та перевірка ролі адміністратора на рівні глобального сховища `useStore`.
- [x] **Protected Routes**: Закриті маршрути адмінки з автоматичним перенаправленням на головну сторінку для неавторизованих користувачів.
- [x] **Audit Log (admin_actions)**: Автоматичне ведення журналу дій адміністратора в Supabase з фіксацією типу операції та деталей.

### 8. UI/UX та Дизайн
- [x] Повністю адаптивна двопанельна навігація (Sidebar на десктопі, гнучка мобільна версія).
- [x] **Animations & Transitions**: Плавна інтеграція анімацій появи, розгортання вікон та ховер-ефектів.
- [x] **Loading States**: Скелетони завантаження таблиць та анімовані індикатори статусу.

### 9. Оптимізація продуктивності
- [x] React Query кешування.
- [ ] **Virtual Scrolling**: Для великих списків користувачів та логів.
- [x] **Debounce**: Оптимізація пошукових запитів.

### 10. Моніторинг та Observability
- [ ] **Sentry**: Відстеження помилок на клієнті та сервері.
- [ ] **Grafana & Prometheus**: Візуалізація системних метрик FastAPI.
- [ ] **Structured Logging**: JSON-логи для зручного аналізу.

### 11. Тестування
- [ ] **Unit Tests**: Тестування логіки (Jest/Pytest).
- [ ] **E2E Tests**: Тестування критичних шляхів (Playwright).

### 12. Додаткові функції
- [ ] **Sync Button**: Кнопка синхронізації коду з GitHub.
- [ ] **Backups**: Автоматичне резервне копіювання БД у хмару.
- [ ] **i18n**: Підтримка української та англійської мов.
- [ ] **Webhooks**: Відправка подій на зовнішні URL.
