# 📐 Архітектура та Технічна специфікація сторінки "Архів" (SvitloSk)

Цей документ визначає архітектурні принципи, модель даних, будову локальної бази даних та UX/UI концепцію для повністю автономного модуля **"Архів"** у клієнтському PWA-додатку **SvitloSk**.

---

## 🎯 1. Архітектурні принципи сторінки

1. **Абсолютна автономність (Isolation)**: Модуль має власну ізольовану директорію `web/src/pages/Archive/` та власну незалежну базу даних на клієнті. Це мінімізує зв'язність коду та дозволяє ізольовано керувати життєвим циклом архівних даних.
2. **Люфт часу (Офсет відображення)**:
   * Дані переносяться в архів з **люфтом в одну добу**.
   * Якщо поточна доба — це день **$D$**, то в архіві відображаються лише стабільні "історичні" дні до **$D - 2$** включно (наприклад, якщо сьогодні 30 травня, то архів доступний до 28 травня включно).
   * Графіки за Сьогодні ($D$) та Вчора ($D-1$) вважаються динамічними (транзитними) та відображаються на Головній сторінці.
3. **Об'єкт операцій — 12 підчерг на добу**:
   * Архів оперує добовими зрізами для всіх 12 підчерг, згрупованих у **6 основних черг** (кожна черга ділиться на 2 підчерги, наприклад: Черга 3 $\rightarrow$ Підчерги 3.1 та 3.2).
   * Інформація за добу по кожній підчерзі візуалізується у вигляді **48-капсульної стрічки** (1 капсула = 30 хвилин).

---

## 🗄️ 2. Специфікація Бази Даних `svitlosk_archive_db` (IndexedDB)

Для збереження автономності ми створюємо окрему базу даних IndexedDB на клієнті: **`svitlosk_archive_db`** (Версія 1) з об'єктним сховищем **`daily_schedules`**.

### Схема документа для запису (`ArchivedDay`):
```typescript
interface ArchivedDay {
  date: string;              // Унікальний ключ у форматі YYYY-MM-DD (Primary Key)
  queues: Record<string, string>; // Зв'язка: "підчерга" (від "1.1" до "6.2") -> 48-значний рядок (напр., "1111000011...")
  meta: {
    savedAt: string;         // ISO Timestamp збереження в БД
    source: "parser" | "manual" | "ocr"; // Джерело даних
    // Перед-розрахована статистика для аналітики та адмінки:
    stats: Record<string, {
      hoursOn: number;       // Загальна кількість годин з освітленням (світло є)
      hoursOff: number;      // Загальна кількість годин без світла
    }>;
  }
}
```

## 🔄 3. Постачання та оновлення даних (Пайплайн)

1. **Первинний старт (Імпорт передісторії)**:
   При першому запуску PWA додаток імпортує невеликий статичний файл з передісторією за останній місяць (`archive_init.json`) безпосередньо у базу даних `svitlosk_archive_db`. Користувач відразу отримує робочий офлайн-архів.
2. **Динамічний дозапис**:
   Коли користувач вибирає дату в календарі:
   * Спочатку перевіряється локальна БД `svitlosk_archive_db`. Якщо дані є — вони відображаються миттєво.
   * Якщо даних немає в локальній БД, додаток робить легкий `fetch` до статичного файлу на сервері `/data/archive/YYYY-MM-DD.json` (розмір ~1.5 КБ).
   * Завантажені дані автоматично зберігаються в IndexedDB для подальшого швидкого офлайн-перегляду.

## 🎨 5. Специфікація відображення: 48-капсульна стрічка

Для кожної підчерги відображається компактна інформаційна картка:
1. **Стрічка з 48 капсул**: Два кольори — **помаранчевий** (світло є) та **сірий** (світла немає). Кожна капсула відповідає 30 хвилинам.
2. **Часова графа (Шкала)**: Мітки часу під стрічкою (`00:00`, `06:00`, `12:00`, `18:00`, `24:00`).
3. **Відбивки змін (Розриви)**: Лаконічний текстовий перелік періодів вимкнень (наприклад: `04:00 - 08:00 (Без світла)`).
4. **Статистика доби**: Лаконічний підсумок: *«Світло: Х год | Без світла: Y год»* (береться безпосередньо з `meta.stats` для максимальної швидкості рендерингу).

## 💎 6. Дизайнерська автономність та візуальні орієнтири

Дотримуючись принципу **абсолютної автономності** (сторінка не імпортує CSS-файли інших розділів і не зв'язана спільними компонентами верстки), візуальне оформлення Архіву створюється як **ізольована локальна реалізація**. Вона підглядає (використовує як орієнтир) візуальну мову Головної сторінки, але відтворює її повністю незалежно.

### Візуальні орієнтири та локальні рішення для Архіву:

* **Концепція Glassmorphism**:
  Календарний блок розробляється як ізольований віджет, що має свій власний локальний стиль плаваючого скла (напівпрозоре тло, розмиття `backdrop-filter: blur(12px)`, тонке срібне обведення). Це створює відчуття преміальності без прямої залежності від стилів `home.css`.
* **Автономні 6 Черг (Рівень 1 гармошки)**:
  Розділи гармошки використовують спеціально створений для архіву CSS-клас `.archive-accordion` (який вже є у `style.css` у виділеному архівному блоці). Вони відтворюють срібний металізований ефект і закруглення кутів (`20px`), але повністю незалежні від інших інтегрованих блоків PWA.
* **48-капсульний таймлайн**:
  Ізольований рендеринг капсул. Сегменти стрічки створюються у вигляді окремих HTML-елементів із заокругленими краями, що використовують локальні кольорові токени: фірмовий помаранчевий та нейтральний темно-сірий. Ніякі інтерактивні елементи шкали часу з Головного екрана не імпортуються.
* **Часова шкала та текстові розриви**:
  Картка підчерги самостійно розраховує та виводить інтервали відключень, базуючись виключно на поточному 48-символьному добовому бітовому рядку. Весь рендеринг є чистим, локальним і не використовує зовнішні хелпери Головної сторінки.
* **Анімація**:
  Для переходів між місяцями та плавного відкриття гармошок використовуються виключно локальні анімації Framer Motion або локальні переходи CSS (`transition: max-height 0.3s ease-out`).
